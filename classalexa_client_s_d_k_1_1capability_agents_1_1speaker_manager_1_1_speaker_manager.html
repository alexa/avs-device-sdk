Detailed Description

This class implementes a CapabilityAgent that handles the AVS Speaker API.

The SpeakerManager can handle multiple SpeakerInterface objects. SpeakerInterface are grouped by their respective types, and the volume and mute state will be consistent across each type. For example, to change the volume of all speakers of a specific type:

// Use local setVolume API.
std::future<bool> result = speakerManager->setVolume(type, AVS_SET_VOLUME_MAX);
// Optionally, wait for the operation to complete.
if (result.valid()) {
result.wait();
}

AVS documentation specifies that Alerts volume should be handled separately. Currently, the AVS API does not support differentiating between different speakers. To provide clients an option to handle Alerts (and any other) volumes independently, only SpeakerInterface::Type::AVS_SYNCED speakers will communicate with AVS.

A default type, SpeakerInterface::Type::LOCAL, is provided. This type will not be modified by directives sent by AVS, nor will they send events on volume/mute change. These SpeakerInterfaces can still be modified through the APIs that SpeakerManagerInterface provides. Clients may extend the SpeakerInterface::Type enum if multiple independent volume controls are needed.

If clients wish directives and events to apply to the specific SpeakerInterface, it must have a type of SpeakerInterface::Type::AVS_SYNCED.
